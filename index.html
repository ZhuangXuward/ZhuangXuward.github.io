前言

实习单位要求做一个单位的官方网站，在做首页的时候有一个轮播图，老生常谈了，我直接就拿之前组对项目的轮播图代码复制粘贴过来。

本来以为是万无一失了，结果出现了之前也出现过的问题：我设置的图片宽度为100%，第一次加载图片通过js获取图片宽度，将图片left平移，移去第一张替补的图片，但是第一次加载时图片似乎多平移了一点距离，导致图片右边出现了一个长条的空白区域，如下图：

而且还好像刚好是滚动条的宽度？

我最初以为是获取元素宽度的问题，以为是多计算了元素滚动区域的宽度，但是转而一想，不对啊，图片根本没有滚动区域啊，而且offsetWidth和clientWidth也只是border-box和padding-box的问题，在这里两者是一样的啊！

等等！“滚动条”！

然后猜想是因为元素加载的问题，这个宽度看起来像是计算了一个宽度W，然后平移的时候，整个网页的宽度变小了，但是平移还是W，所以才会多出那一长条！多出来的是滚动条，是因为一开始计算宽度的时候根本就没有滚动条，之后因为网页内容出现了滚动条，就导致了获取宽度和加载完毕宽度不一致的情况。

mouseover和mouseenter的区别

mouseover和mouseenter

两者的不同通过两点解释：

  • 是否支持冒泡
  • 事件触发时机

查找了一些资料，有了一个比较清晰的解释：

  1. mouseover支持冒泡，当鼠标从元素边界之外进入元素时被触发；当鼠标在元素边界内时，进入元素的子元素，也会触发mouseover事件；
  2. mouseenter不支持冒泡，同样鼠标从边界之外进入时触发；但由于不支持事件冒泡，当鼠标在元素边界内，要再次触发时，必须将鼠标移到边界外，再次移入才可触发。

也就是说，两者的区别表现在当鼠标处于选定元素边界内时，鼠标进入子元素是否触发。

因此，当鼠标在元素子元素进入或离开时，只会触发mouseover和配对的mouseout，而不会触发mouseenter和mouseleave。

在线演示：点击进入

动图如下:
鼠标指针穿过被选元素或其子元素，都会触发 mouseover 事件。

只有在鼠标指针穿过被选元素时，才会触发 mouseenter 事件。

原理解释

添加一个类构造函数来初始化状态 this.state，类组件应始终使用 props 调用基础构造函数。

每当 Clock 组件第一次加载到 DOM 中的时候，我们都想生成定时器，这在 React 中被称为挂载。

同样，每当 Clock 生成的这个 DOM 被移除的时候，我们也会想要清除定时器，这在 React 中被称为卸载。

componentDidMount()和componentWillUnmount()方法被称为生命周期钩子，在组件输出到DOM后会执行componentDidMount()，我们就可以在这个方法中设置定时器，this.timerID是定时器的ID。而组件被从DOM中移除时，React就调用componentWillUnmount()方法。

使用对象的属性来设置定时器ID能免除全局和局部变量的作用域问题。

几个方法的执行顺序：

  1. 当 被传递给 ReactDOM.render() 时，React 调用 Clock 组件的构造函数。 由于 Clock 需要显示当前时间，所以使用包含当前时间的对象来初始化 this.state 。 我们稍后会更新此状态。

  2. React 然后调用 Clock 组件的 render() 方法。这是 React 了解屏幕上应该显示什么内容，然后 React 更新 DOM 以匹配 Clock 的渲染输出。

  3. 当 Clock 的输出插入到 DOM 中时，React 调用 componentDidMount() 生命周期钩子。 在其中，Clock 组件要求浏览器设置一个定时器，每秒钟调用一次 tick()。

  4. 浏览器每秒钟调用 tick() 方法。 在其中，Clock 组件通过使用包含当前时间的对象调用 setState() 来调度UI更新。 通过调用 setState() ，React 知道状态已经改变，并再次调用 render() 方法来确定屏幕上应当显示什么。 这一次，render() 方法中的 this.state.date 将不同，所以渲染输出将包含更新的时间，并相应地更新 DOM。

  5. 一旦 Clock 组件被从 DOM 中移除，React 会调用 componentWillUnmount() 这个钩子函数，定时器也就会被清除。

之前还在想，“不就是显示个时间嘛，干吗要大刀阔斧用React？JS不是更快捷？”现在这个疑问算是解决了，我从这个疑问中感受到React的实际作用。

offsetWidth、getBoundingRect()、scrollWidth、clientWidth用法总结

offsetWidth

offsetWidth是一个只读属性，返回一个元素的布局宽度，为border-box的宽度。

语法：var offsetWidth =element.offsetWidth; 结果为四舍五入的整数。


getBoundingClientRect()

offsetWidth和其他方式返回的是四舍五入（round）的整数，想要得到小数则要用getBoundingClientRect()方法。

getBoundingClientRect()返回元素的大小及其相对于视口左上角的位置。

包括的属性有：width、height、left、right、top、bottom。


clientWidth

表示元素的内部宽度，以像素计。该属性包括内边距，但不包括垂直滚动条（如果有）、边框和外边距。


scrollWidth

表示元素内容的宽度，包括由于滚动而未显示在屏幕中内容。

宽度的测量方式与clientWidth相同：它包含元素的内边距，但不包括边框，外边距或垂直滚动条（如果存在）。
